# CHM Generator WPF - 快速开始

## 🚀 首次使用配置

### 1. 配置 Python doc2html（可选但推荐）

如果你想使用功能更强大的 Python 转换器：

1. **打开设置窗口**
   - 点击主界面顶部工具栏的 `⚙ 设置` 按钮

2. **启用 Python 转换器**
   ```
   ☑ 使用 Python doc2html 转换 Word 文件（推荐）
   ```

3. **选择 Python 工具目录**
   - 点击 `浏览...` 按钮
   - 选择你的 Python 项目目录（如：`D:\python\doc2html`）
   - 确保该目录包含：
     - `DocToCHM.py` ✅
     - `hyperlink_processor.py` ✅

4. **检查状态**
   - 状态栏显示：`✓ 已启用，工具文件完整` 表示配置成功

5. **保存配置**
   - 点击 `保存` 按钮

**如果不配置：** 程序会自动使用内置转换器（OpenXmlPowerTools），功能也完全可用。

## 📖 基本使用流程

### 步骤 1：添加文件

**方式一：点击按钮**
- 点击 `📂 添加文件` - 选择 HTML 或 Word 文档
- 点击 `📁 添加文件夹` - 批量导入整个文件夹
- 点击 `➕ 新建文件夹` - 创建虚拟文件夹组织结构

**方式二：拖拽**
- 直接从资源管理器拖拽文件到左侧文档树

### 步骤 2：组织文档结构

- **拖拽排序**：直接拖动节点调整顺序和层级
- **上移/下移**：使用工具栏按钮或右键菜单
- **重命名**：双击节点标题或按 F2 键
- **删除**：选中后按 Delete 键或点击删除按钮

### 步骤 3：设置 CHM 标题

在顶部工具栏的输入框中输入 CHM 标题（如："帮助文档"）

### 步骤 4：生成 CHM

1. 点击 `🔨 生成 CHM` 按钮
2. 选择输出位置（父目录）
3. 程序自动创建项目文件夹（基于 CHM 标题）
4. 等待转换和编译完成

**输出结构：**
```
选择的目录/
└── 帮助文档/              ← 自动创建
    ├── project.hhp
    ├── toc.hhc
    ├── index.hhk
    ├── 帮助文档.chm       ← 最终生成的 CHM 文件
    └── src/               ← HTML 源文件
```

## 🔧 常见场景

### 场景 1：纯 HTML 文件生成 CHM

1. 添加 HTML 文件
2. 调整顺序
3. 点击生成 CHM

**优点：** 速度快，直接使用

### 场景 2：纯 Word 文档生成 CHM

1. 添加 Word 文档（.docx）
2. 程序自动转换为 HTML
3. 点击生成 CHM

**两种转换方式：**
- **Python doc2html**（需配置）：格式保留更好，支持复杂表格和脚注
- **内置转换器**（自动回退）：无需配置，自动可用

### 场景 3：混合使用 HTML 和 Word

1. 添加 HTML 文件（如首页）
2. 添加 Word 文档（如各个章节）
3. 创建文件夹组织层级
4. 点击生成 CHM

**所有内容会自动合并到一个 CHM 中！**

## ⚙️ 高级选项

### CHM 编译选项（右下角展开器）

- **全文搜索**：启用 CHM 内部搜索功能
- **二进制 TOC**：生成二进制目录（推荐）
- **自动索引**：自动生成索引条目

### 文件夹自动命名

- 程序根据 CHM 标题自动创建文件夹
- 如果已存在，自动添加序号（如 `帮助文档_1`）
- 避免覆盖现有项目

## 🐛 故障排除

### 问题 1：Python 转换失败

**症状：** 日志显示 "Python 转换失败"

**解决方案：**
1. 检查 Python 是否已安装：`python --version`
2. 检查依赖包：`pip install python-docx pillow lxml`
3. 检查设置窗口中的路径是否正确
4. **不用担心：** 程序会自动回退到内置转换器

### 问题 2：CHM 编译失败

**症状：** 日志显示 "未找到 hhc.exe"

**解决方案：**
1. 安装 **Microsoft HTML Help Workshop**
2. 或点击工具栏的 `🔧` 按钮手动指定 hhc.exe 位置
3. 下载地址：[Microsoft HTML Help Downloads](https://learn.microsoft.com/en-us/previous-versions/windows/desktop/htmlhelp/microsoft-html-help-downloads)

### 问题 3：CHM 无法打开或显示空白

**可能原因：**
- HTML 文件编码问题（需要 GB2312）
- 文件名包含特殊字符
- 路径过长

**解决方案：**
- 使用 Python doc2html（自动处理编码）
- 避免使用特殊字符命名文件
- 缩短输出路径

## 📝 配置文件位置

**配置保存在：**
```
%AppData%\CHMGeneratorWPF\config.json
```

**内容：**
```json
{
  "UsePythonConverter": true,
  "PythonToolsPath": "D:\\python\\doc2html"
}
```

**重置配置：** 删除该文件即可恢复默认设置

## 🎯 最佳实践

### 1. 文件命名
- 使用有意义的文件名（会成为 CHM 中的标题）
- 避免使用特殊字符：`<>:"/\|?*`

### 2. 文档组织
- 使用文件夹建立清晰的层级结构
- 首页放在最上面
- 相关主题放在同一文件夹

### 3. Word 文档格式
- 使用标题样式（标题 1、标题 2）建立大纲
- 图片尽量使用 PNG 或 JPEG 格式
- 避免使用过于复杂的表格

### 4. 测试流程
- 生成 CHM 后先预览
- 检查目录结构是否正确
- 测试搜索功能是否可用
